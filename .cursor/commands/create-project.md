# /create-project 命令

## 命令格式
```bash
/create-project [一段描述性的话]
```

## 功能说明
创建新的Spring Boot项目，包括项目目录结构、基础配置文件、依赖管理、项目文档和基础代码模板。

## AI分析流程

### 1. 智能描述解析
- **提取关键信息**：从用户描述中提取项目名称、功能需求、技术需求
- **识别项目类型**：判断是管理系统、API服务、微服务等
- **推断技术栈**：基于描述推断合适的技术选型

### 2. 智能文档发现
- **自动扫描**：扫描项目根目录下的`docs/`目录中的需求文档、架构文档、技术文档
- **内容匹配**：基于描述内容匹配相关文档
- **信息提取**：从文档中提取项目需求、架构设计、技术选型等
- **文件显示**：在AI响应中显示找到的相关设计文件名，便于用户确认

### 3. 基于项目类型的智能推断
- **项目分析**：分析项目描述推断项目类型和复杂度
- **技术推断**：基于项目类型推断合适的技术栈
- **架构设计**：推断项目的整体架构和设计模式

### 4. 项目结构规则匹配
- **规范匹配**：根据项目类型匹配对应的项目结构规范
- **最佳实践**：应用Spring Boot项目的最佳实践
- **配置生成**：生成符合项目需求的配置文件

### 5. 主动询问机制
- **需求补全**：识别缺失或模糊的需求，主动询问用户
- **技术确认**：确认技术选型和架构设计
- **多轮对话**：支持多轮对话，逐步完善项目需求

## 使用示例

### 1. 简单描述
```bash
/create-project 学生管理系统
```

### 2. 详细描述
```bash
/create-project 学生管理系统，包含用户管理、课程管理、成绩管理、选课管理，支持多角色权限控制
```

### 3. 业务场景描述
```bash
/create-project 电商系统，包含商品管理、订单管理、支付管理、用户管理，支持移动端API
```

### 4. 技术需求描述
```bash
/create-project 博客系统，包含文章管理、评论管理、用户管理，需要支持Markdown编辑和图片上传
```

## 生成内容

### 1. 项目结构
- 标准的Spring Boot项目目录结构
- 符合规范的分层架构
- 标准的包命名结构

### 2. 配置文件
- `build.gradle` - Gradle构建配置
- `application.yml` - 主配置文件
- `application-dev.yml` - 开发环境配置
- `application-test.yml` - 测试环境配置
- `application-prod.yml` - 生产环境配置

### 3. 基础代码
- `Application.java` - 主启动类
- `GlobalExceptionHandler.java` - 全局异常处理
- `CommonResponse.java` - 统一响应包装类
- `CommonPageResponse.java` - 分页响应包装类

### 4. 项目文档
- `README.md` - 项目说明文档
- `.gitignore` - Git忽略文件配置

## 项目特性
1. **标准化结构**: 遵循Spring Boot最佳实践
2. **分层架构**: Controller、Service、Mapper分层设计
3. **配置管理**: 多环境配置支持
4. **异常处理**: 全局异常处理机制
5. **统一响应**: 标准化API响应格式
6. **代码规范**: 符合项目编码规范
